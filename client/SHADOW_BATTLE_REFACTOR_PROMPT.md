# Промт: рефакторинг ShadowBattleScene — полноценный ИИ-пилот

**Файл:** `client/src/scenes/ShadowBattleScene.js`

---

## Контекст

Режим "Бой с тенью" — это 1v1 PvP-арена, где противник — **не моб**, а корабль с теми же механиками что у игрока: статы корабля + уровень скиллов + расходники + панель действий. Единственное отличие — пилотирует его ИИ. Никаких `mob.passive`, `leash`, `bossRef`, `aiClass`, `scatter shot` или любой другой mob-механики быть не должно.

---

## Модель "бота" (чего нет сейчас, что должно быть)

Бот — это структура `BotPilot` с теми же полями что у игрока в GameScene:

```js
{
  // Корабль
  shipDef, shipLevel, activeShip,
  hull, maxHull, shield, maxShield,
  speed, baseSpeed,

  // Оружие (из equipped модулей, тир + тип)
  weaponDamage, weaponPenetration, weaponType,  // 'plasma'|'laser'
  fireRate, fireCd,

  // Скиллы — те же ключи что у игрока (overcharge_shot, salvo, shield_burst, emergency_repair, stealth_sprint, berserker)
  skillLevels: { overcharge_shot: 3, salvo: 2, ... },
  skillCooldowns: {},   // аналог gs.skillCooldowns
  skillCooldownMs(key), // аналог gs._skillCooldownMs(key)

  // Расходники — берутся из конфига арены (не из inventory игрока)
  inventory: [ { type:'repair_pack', qty:2 }, { type:'speed_boost', qty:1 } ],

  // Движение
  x, y, vx, vy, heading,
  waypoint: null,
  speedMult: 1,

  // Статусы (те же что у игрока)
  overchargeActive: false,
  invincible: false, invincTimer: 0,
  berserkerMult: 1, berserkerTimer: 0,
}
```

## Конфиг перед боем

Пользователь выбирает параметры бота (уже есть в текущей версии):
- модель корабля (карусель по `BATTLE_SHIPS`)
- уровень корабля (1–10)
- тир снаряжения (T1–T4)
- тип оружия (plasma / laser)
- уровень пилота (1–50) — определяет уровни скиллов бота (линейно: `Math.floor(pilotLevel / 50 * maxSkillLevel)`)

Из этого конфига собирается `BotPilot` — один раз при старте арены.

## ИИ-пилот: логика принятия решений

ИИ работает в `update(dt)` и принимает решения на основе дистанции и состояния боя. **Никаких mob-паттернов.** Только поведение, которое мог бы исполнить человек-пилот:

### Движение (state machine)

```
dist > 900px  → approach: двигаться к игроку на полной скорости
300–900px     → strafe: двигаться перпендикулярно к игроку, менять сторону каждые 1.2–2.0с
< 300px       → retreat: отступать от игрока
hull < 20%    → flee: отступать на максимальной скорости, приоритет выше strafe
```

**Уклонение от снарядов:** если снаряд игрока ближе 180px — краткосрочный перпендикулярный рывок (0.4с). Проверка каждый кадр по массиву `this._projs`.

### Авто-огонь

`fireCd -= dt`, при `fireCd <= 0` — выстрел в направлении игрока с предикторным прицелом — упреждение на `playerSpeed * dist / projSpeed`.

### Использование скиллов (ИИ-приоритеты, проверка раз в 0.5с)

```
hull < 30% && emergency_repair готов      → использовать
hull < 30% && repair_pack есть            → использовать
hull < 50% && shield_burst готов          → использовать
dist < 400px && stealth_sprint готов      → использовать (уклонение)
dist < 600px && overcharge_shot готов     → использовать (следующий выстрел ×2)
dist < 800px && salvo готов               → использовать
hull > 50% && berserker готов             → использовать
hull < 15% && speed_boost есть            → использовать (побег)
```

## Снаряды — единая система

Должна быть **одна** функция `_fireProjectile(owner, fromX, fromY, toX, toY, weaponType, damage, pen)`:

- `owner: 'player' | 'bot'`
- `weaponType: 'plasma' | 'laser' | 'ion'` (ion — T4 plasma, 3 болта в веере)
- `laser` → мгновенный `void`-луч (линия + урон сразу)
- `plasma` / `ion` → projectile-объект в `this._projs[]`
- Урон в `_applyHit(target, damage, pen)` где `target: 'player'|'bot'`

## HUD

- Снизу: панель действий игрока — без изменений.
- Снизу справа (или сверху справа): **панель действий бота** — визуальная (не кликабельная), показывает иконки скиллов бота с КД-оверлеем. Позволяет игроку читать состояние противника.
- Сверху: полоски HP/Shield игрока (слева) и бота (справа) — уже есть.

## Награды

- **XP и кредиты** — всегда при победе
- **Честь** — только если `botPower > playerPower`
- `powerRating` считается одинаковой формулой для обеих сторон по финальным статам

## Карта и окружение — идентичны обычной игре

Арена боя **не должна выглядеть как отдельная сцена с сеткой и стенами**. Визуально и инструментально это та же игровая среда что и обычная карта:

- **Фон**: тот же параллакс-звёздный фон (`BackgroundScene` запущена параллельно, как в GameScene)
- **HUD**: полноценный HUD (`HudScene` активна) — мини-карта, статы, панель действий, event log — всё работает как обычно. Никаких отдельных самодельных полосок HP поверх сцены
- **Управление**: то же что в GameScene — клик мышью задаёт waypoint, корабль игрока движется с той же физикой (`Player.js` или аналогичная логика), авто-огонь по ближайшей цели
- **Инструменты**: спринт, скиллы, расходники — через ту же панель действий HudScene, те же горячие клавиши

### Реализация через GameScene, а не отдельную сцену

Конфиг-панель — отдельный оверлей (HTML или Phaser overlay), запускается поверх обычного игрового экрана. После нажатия "НАЧАТЬ БОЙ":

1. Конфиг-панель закрывается
2. Игрок телепортируется на персональный сектор `shadow_arena` — обычный сектор в `galaxy.js` с флагом `{ personal: true, pvp: false, isDungeon: false }`
3. Сектор пустой (никаких мобов, лута, базы) — только игрок и один бот-корабль
4. Бот спавнится как обычный объект на сцене (не через `Mob.js`), управляется `BotPilot` логикой
5. По завершению боя (победа/поражение) — обычный возврат на предыдущий сектор через jumpgate-механику или прямой телепорт назад

### Размер и границы арены

- Размер карты: такой же как стандартный сектор в `SECTORS` (`map: 'sector_default'` или аналог)
- Границы — те же world bounds что в GameScene, без видимых стен
- На мини-карте в HUD видно позицию игрока и бота (бот отображается как красная точка, аналог вражеского игрока в PvP)

### Что из текущего `ShadowBattleScene` нужно перенести в GameScene

| Что | Куда |
|---|---|
| `buildBotPilot(cfg)` | `GameScene` — создаётся при входе в `shadow_arena` |
| Логика `BotPilot.update(dt)` | `GameScene.update()` — после `player.update()`, до `mobs` |
| Конфиг-панель | Отдельный overlay-компонент, открывается из HudScene nav-бара |
| Результирующая панель (победа/поражение) | Phaser overlay поверх GameScene, кнопка "НА БАЗУ" делает `_exitToSpace()` |

`ShadowBattleScene.js` после рефакторинга может быть удалён или оставлен только как тонкий контроллер конфиг-панели без собственной карты.

---

## Итог

После рефакторинга бот должен ощущаться как **реальный противник-пилот**: предсказывает выстрел, уклоняется, использует скиллы осмысленно в нужный момент. Никакой разницы в базовых механиках между ботом и игроком — разница только в том, кто принимает решения. Никакой разницы в визуале и инструментах между ареной и обычной игровой картой — это та же среда, просто персональный пустой сектор с одним противником.
