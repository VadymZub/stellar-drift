# Динамическое масштабирование PvP секторов

## Описание
Для создания ощущения масштабности в боевых зонах реализована система динамического изменения размеров игрового мира в зависимости от типа сектора. PvP-секторы имеют площадь в 4 раза больше стандартных (линейное увеличение сторон в 2 раза).

## Техническая реализация

### 1. Константы (`src/constants.js`)
Вместо статичного объекта `WORLD` введены базовые параметры и коэффициент масштабирования:
- `BASE_WORLD`: Базовые размеры (6929x3900).
- `PVP_WORLD_SCALE`: Коэффициент `2.0` для PvP секторов.

### 2. Инициализация сцены (`src/scenes/GameScene.js`)
В методе `create()` выполняется проверка флага `pvp` текущего сектора:
```javascript
const sec = SECTORS[galaxy.current];
const isPvp = sec.pvp === true;
const scale = isPvp ? PVP_WORLD_SCALE : 1.0;

this.worldWidth = BASE_WORLD.width * scale;
this.worldHeight = BASE_WORLD.height * scale;
```

### 3. Границы и позиционирование
- **Bounds**: Камера и физика получают динамические границы `(0, 0, this.worldWidth, this.worldHeight)`.
- **Центрирование**: База, джампгейты и точки спавна мобов рассчитываются относительно `this.worldWidth / 2` и `this.worldHeight / 2`.
- **Переходы**: При прыжке через врата (`travelTo`) координаты появления (`startX/startY`) пересчитываются с учетом масштаба целевого сектора.

## Результаты
- Обычный сектор: **6929 x 3900** px.
- PvP сектор: **13858 x 7800** px.
- Объекты (база, мобы) сохраняют корректное относительное положение в центре карты.
