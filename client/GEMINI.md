# Stellar Drift — Инструкции и Правила Проекта

Этот файл является фундаментальным руководством для Gemini CLI при работе над проектом. Инструкции здесь имеют приоритет над общими правилами.

## 1. Технологический стек
- **Движок**: Phaser v4.1.0 (ESM-модули через CDN).
- **Архитектура**: No-build (запуск через `python -m http.server`).
- **Язык**: JavaScript (ES6+).

## 2. Критические API Phaser 4 (Важно!)
Phaser 4 существенно отличается от Phaser 3. **НЕ ИСПОЛЬЗОВАТЬ** устаревшие методы:
- **Частицы (Particles)**:
    - НЕТ метода `setSpeed()`, `setAngle()`, `setScale()`.
    - ИСПОЛЬЗОВАТЬ прямой доступ к свойствам эмиттера: `emitter.speedX`, `emitter.speedY`, `emitter.scaleX`, `emitter.scaleY`, `emitter.particleRotate`.
    - Для запуска/остановки: `emitter.emitting = true/false`.
- **Сцены**:
    - Переход через `this.scene.restart({ startX, startY })` для смены секторов.

## 3. Архитектурные Соглашения
### Сущности (Entities)
- **Player.js**:
    - Флаг `lockedRotation`: если `true`, `update()` игнорирует изменение `facing`. Обязательно включать во время анимаций прыжка.
    - `facing`: угол носа корабля. К нему всегда прибавлять `ART_ANGLE_OFFSET` (90°) при записи в `sprite.rotation`.
- **Mob.js**:
    - Метод `update()` принимает аргументы: `(dt, player, inSafeZone, fireCallback)`.

### Геймплей и Управление
- **Бой**:
    - Только ручная стрельба. Флаг `isFiring` в `GameScene` переключается через `CTRL` или дабл-клик по мобу.
    - Дабл-клик по мобу: автоматический захват цели (`selectTarget`) + установка `isFiring = true`.
    - При прыжке/смерти `isFiring` должен сбрасываться в `false`.
    - В `Player.js` обязательно наличие `this.weaponRange` для работы проверок дистанции в `GameScene`.
- **Прыжки (Jumpgates)**:
    - Обязательная проверка `sectorAccess` из `galaxy.js` перед началом анимации.
    - Вихрь (`vortex`) скрыт по умолчанию. Появляется только при старте `startJumpSequence`.
    - Для масштабирования вихря использовать `displayWidth/displayHeight` (цель ~160px), а не `scale`.
    - Последовательность прыжка: Разворот носом (0.4с) -> Раскрутка вихря + Сжатие корабля (2.7с) -> Вспышка (0.3с) -> `restart`.

## 4. Визуальные стандарты
- **Глубина (Depth)**:
    - Звёздная пыль (`dust`): -10 (должна быть видна над фоном).
    - Фон: -19.
    - Джампгейт (кольцо): 4 (должно перекрывать вихрь).
    - Вихрь: 2.
    - Ретикл (захват цели): 45.
- **Эффекты скорости**:
    - Частицы `dust` отображаются **ТОЛЬКО** во время форсажа (boost). При обычном полете они скрыты.
    - Направление движения частиц должно **СОВПАДАТЬ** с направлением движения корабля (лететь вперед).
    - При ускорении частицы должны вытягиваться по оси X (`scaleX`) в 8.0-12.0 раза, создавая эффект "forward warp streaks".
    - Базовая яркость пыли (`alpha`): 0.8.

## 6. Протокол безопасности (Safety First)
- **Запрет на удаление**: ПЕРЕД удалением любой строки кода проверить её использование в текущем методе и смежных файлах через `grep_search`.
- **Peer Review**: Для сложных правок (например, боевая система или физика) вызывать sub-agent `generalist` с задачей "Проверь план изменений на предмет регрессий".
- **Валидация**: Перед внесением правок в `GameScene.js`, проверить сигнатуры методов в `Player.js` или `Mob.js`.
- **Phaser 4 API Only**: Никаких `setSpeed`, `setScale` и т.д. Только свойства объекта.
